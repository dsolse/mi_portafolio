Ideación de idea de negocios con design thinking

Ideación de Chubby, un servicio entregado mediante una app para restaurantes locales emprendedores a fin de publicar sus platillos y usar algoritmos para mostrar sus platillos a clientes potenciales acorde a sus gustos. Se usó la metodología design-thinking siguiendo sus cuatro pasos para crear una idea de negocio a partir de una necesidad: empatizar con usuarios disgustados por la monotonía de sus opciones de comida y su poco tiempo para buscar nuevas opciones acorde a sus preferencias; después se procedió a definir el problema estudiando más a fondo a los usuarios en busca de sus necesidades; a continuación, se idea la solución viendo las mejores alternativas, y finalmente, hacemos un prototipo que testeamos con el usuario de manera iterativa hasta satisfacer sus expectativas y solucionar su problema. Al final la solución fue un asistente de voz con previsualización de platillos en 3D y algoritmos y filtros para ajustarse al cliente con un modelo de monetización de micro transacciones y previsualización de las ganancias hasta cinco años.

A lo largo del proyecto, se uso la metodología design-thinking, pues permie partir desde un problema latente hasta construir una idea de negocio innovadora llevadera al corto, medio y largo plazo.

Evaluado por Licda. Maria Jose Calderone de Ramírez

Codificador de mensajes con python

Creación de una aplicación de consola de Python para codificar y decodificar mensajes. El proyecto se basó programación orientada a objectos, se creó una clase que contiene dos métodos, uno para encriptar y otro para desencriptar otro mensaje; el funcionamiento de la aplicación era reemplazar cada carácter con otro que era equivalente al anterior pero desplazado cualquier cantidad de posiciones. Asimismo, se automatizó el programa a fin de agregar los archivos a trabajar en una carpeta, usando las funciones de Python para manipular directorios del sistema operativo.

Con el proyecto se evaluó el conocimiento de Python aprendido a lo largo del curso aplicándolo para una aplicación que involucrara buenas prácticas y lógica de programación.

Evaluado por catedrático Alejandro Alas

Implementación de mínimos cuadrados con álgebra lineal

Resolución de caso de estimación de un bien usando los conocimientos adquiridos en álgebra lineal usando los mínimos cuadrados. El proyecto consistió en trabajar un conjunto de datos de muestra para deducir una función que dado la cantidad de activos de un bien estime su capital, usando herramientas como Excel y la factibilidad que ofrece operar matrices con el software. Se procedió a hacer estimaciones con funciones lineales, cuadradas y cúbicas, para al final comparar cuál tenía un menor margen de error entre el valor real y el estimado.

Elavuado por Licda. Roxana Palacios

Propuesta de renovación de servicio siguiente el service design

Se propuso una propuesta al taller Taller Palacios sobre la renovación de su proceso de tracking cuando se lleva una reparación. Se siguió la metodología service-design para ahondar en cada punto de dolor del usuario comenzando con conocer a los usuarios principales y la situación del servicio el cual no daba abasto a los clientes y no cumplía su función principal: fidelizar con los clientes y atraer a nuevos. Después se planteó el orden más eficiente del proceso y se comparó con el actual. Después se entrevistó a usuarios y se iteró hasta mejorar la propuesta. Entre las soluciones se dieron la implementación de tecnologías fáciles de usar como Excel, WhatsApp empresarial, reseñas en Instagram y un rediseño del servicio enfocado en el aprendizaje del sobre el estado de su vehículo para darle seguridad y atender a sus exigencias.

Con este proyecto se dio un uso exhaustivo del service-design aplicando sus principios enfocados en el usuario y su experiencia.

Evaluado por catedrática Jessica Ponce

Propuesta de renovación de proceso

Propuesta a la compañía Le Croissant de mejora de proceso llamado formulación, el cual consiste en la creación de la masa para los panes, el cual tenía inconvenientes como retrasos o mala elaboración de la masa y, por consiguiente, su desperdicio. Para trabajar el proceso usamos múltiples herramientas y la colaboración de uno de los encargados de los procesos. Se dio uso del diagrama causa-efecto para ver las causas de los defectos, los ocho desperdicios a modo de ver de manera más puntual los posibles desperfectos y otras como el DMAIC que ayudó a buscar las soluciones que mejoraban la eficiencia. Pudimos observar que la masa era responsabilidad del departamento de bodega que, junto con sus respectivas obligaciones, tenía que hacer una labor que le compete a producción, encargado de hacer los panes con la masa, lo cual reajustamos dando sus tareas a cada lado. Asimismo, con un archivo de Excel se creó un prototipo de un software que ayuda a la coordinación de los departamentos, desde compras hasta entregas.

Evaluado por catedrática María Alejandra Grisales Albarracín

Lector de formato epub

Creación de lector ebook (solo soporta epub) usando framework de desarrollo multiplataforma flutter, enfocado para aprendices de idiomas. La app consiste en un asistente para guardar palabras desconocidas y mostrarlas resaltadas en cada libro para poder clickearlas y ver su definición. Las palabras y los datos de los libros quedan guardados en una base de datos Sqlite3. Asimismo, trae un menú al seleccionar el texto, diccionario y visualización del significado, usando un componente webview. Asimismo, permite cambios de fuente y estilo y una ventana extra para tener un texto extra y así formar un lector de libros paralelos (puede ser la tradución).
Hay una versión de solo android nativo bajo desarrollo, pero ya implementa las mismas características de la original
